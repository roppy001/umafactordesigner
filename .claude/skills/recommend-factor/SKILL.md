# 因子構成推奨

`references/trainee.json` と `references/racetrack.json` を読み込み、
指定のレース・距離・脚質に対する推奨親・祖父母構成を `output/` に出力する。

## 入力パラメータの読み取り

ユーザーの発言から以下を抽出する:

| パラメータ | 例 | 必須 |
|-----------|---|------|
| レース場 | `東京` | ✅ |
| 路面 | `芝` | ✅ |
| 距離（m） | `2400` | ✅ |
| レースモード | `チャンミ` または `LoH` | ✗（デフォルト: `チャンミ`） |
| 制約条件 | `親は加速持ちにしたい` | ✗ |

距離が省略された場合はユーザーに確認する。
路面が省略された場合は `芝` をデフォルトとしてよい。
レースモードが省略された場合は `チャンミ`（9頭立て）をデフォルトとする。

レースモードからフィールドサイズを導出する:

| race_mode | field_size |
|-----------|-----------|
| `チャンミ` | 9 |
| `LoH` | 12 |

## 実行手順

### ステップ1: 参照データの読み込み

`references/racetrack.json` と `references/trainee.json` を読み込む。
どちらかのファイルが存在しない場合は処理を中止し、対応するスキルを先に実行するよう案内する。

- racetrack.json がない → 「レース場情報を保存してください」と案内
- trainee.json がない → 「継承固有の情報を保存してください」と案内

### ステップ2: 対象コースの特定

`racetrack.json` から `venue`・`surface`・`distance_m` が全て一致するコースを検索する。

- 一致するコースが見つからない場合: 同じ `venue` と `surface` で存在するコース一覧を距離順に提示して処理を中止する
- 一致するコースが複数ある場合: 全て対象とする（通常は1件）

### ステップ3: スキルのスコアリング

`trainee.json` の全スキルに対して、以下の観点でスコアを付ける。
スコアは加算方式（最大100点）とし、各脚質ごとに独立して計算する。

#### 3-0. rank_range の参照（レースモード対応）

`activation_conditions.rank_range` はモード別に事前格納済み。`race_mode` に対応するキーを直接参照する:

```
effective_rank_range = rank_range?.[race_mode]   // "チャンミ" or "LoH"
```

- `rank_range` が `null` の場合: 順位条件なし（除外しない）
- `effective_rank_range[0] > field_size` の場合: このフィールドでは発動不能 → **スコア 0 点で候補除外**
- `effective_rank_range[1]` が null の場合: 上限なし扱い（除外しない）

例（race_mode = "チャンミ", field_size = 9 の場合）:
- `{ "チャンミ": [7, 9], "LoH": [7, 12] }` → 実効 `[7, 9]`（有効・後方スキル）
- `{ "チャンミ": null, "LoH": [10, 12] }` → 除外（9頭立て非対応）
- `{ "チャンミ": [1, 3], "LoH": [1, 3] }` → 実効 `[1, 3]`（変化なし・前方スキル）

**rank_multiplier の計算:**

スキルが除外されない場合（`rank_range` が null、または有効な `effective_rank_range` を持つ場合）、以下のように `rank_multiplier` を決定する。この値は 3-1〜3-5 で算出した合計スコアに最後に乗算する。

```
rank_range が null の場合:
  rank_multiplier = 1.0

rank_range が非 null かつ有効な場合:
  range_min = effective_rank_range[0]
  range_max = effective_rank_range[1] ?? field_size   // null は field_size で代替
  prob_table = 上記「順位確率分布表」の race_mode × 対象脚質の行
  rank_multiplier = sum(prob_table[range_min], prob_table[range_min+1], ..., prob_table[range_max])
```

- 対象脚質: 現在スコアリング中の脚質（逃げ / 先行 / 差し / 追込）を使用する
- `running_styles` が `[]`（脚質不問）のスキルも、スコアリング対象脚質の行を参照する
- 確率分布表がすべて 0.00 の場合（ひな形未入力）: `rank_multiplier = 1.0` として扱い、乗算をスキップする

例（先行・チャンミ・rank_range = [3, 4]、表の先行3位=0.25・4位=0.25 の場合）:
```
rank_multiplier = 0.25 + 0.25 = 0.50
最終スコア = (3-1〜3-5 の合計点) × 0.50
```

#### 3-1. 脚質フィルター（必須条件）

`activation_conditions.running_styles` が空配列 `[]` または対象脚質を含む場合のみスコア計算対象とする。
含まない場合はその脚質での推奨候補から除外する。

#### 3-2. フェーズ・距離スコア（最大40点）

スキルの発動距離% と コースのフェーズ区間% を照合する。

コースのフェーズ区間を% に変換:
- `late` フェーズ%: `phases.late.start_m / distance_m * 100` 〜 `phases.late.end_m / distance_m * 100`
- `last_sprint` フェーズ%: `phases.last_sprint.start_m / distance_m * 100` 〜 100

スキルの発動区間 `[distance_pct_min, distance_pct_max]` との重複を判定:

| 重複条件 | 加点 |
|---------|-----|
| `last_sprint` フェーズと重複 | +40点 |
| `late` フェーズのみと重複 | +25点 |
| `middle` フェーズ（33%〜67%）のみと重複 | +10点 |
| `distance_pct_min/max` が両方 null | +10点（フェーズ不問スキル） |

#### 3-3. 効果タイプスコア（最大30点）

`inherited_effects` の効果種別に応じて加点:

| 効果 | 加点 |
|-----|-----|
| `acceleration_mps2` が null でない | +30点 |
| `target_speed_mps` が null でない | +20点 |
| `stamina_recovery_pct` が null でない | +10点 |

複数効果がある場合は最大の1つのみ加点する。

#### 3-4. コース特性一致スコア（最大20点）

コースの `skill_aptitudes` リストと、スキルの `skill_name` または発動条件の内容を照合:

| 条件 | 加点 |
|-----|-----|
| コースの `direction`（右/左）と `skill_aptitudes` に「右回り」/「左回り」が含まれ、スキル名にも合致する語句がある | +20点 |
| コースに坂があり、スキル名に「坂」「上り」「下り」を含む | +15点 |
| `skill_aptitudes` にレース場固有スキル名があり、スキル名に一致する | +10点 |

#### 3-5. 継承コストペナルティ（最大 −10点）

`inherited_effects.cost_pt` が高いほど優先度を下げる:

| コスト | 補正 |
|-------|-----|
| ≤ 150pt | 0点 |
| 151〜200pt | −5点 |
| ≥ 201pt | −10点 |

#### 3-6. ユーザー制約条件による絞り込み

制約条件が指定されている場合、以下のルールで候補を絞り込む（絞り込み後に候補がゼロになる場合は絞り込みを緩和して警告を出す）:

| 制約キーワード例 | 適用ルール |
|---------------|----------|
| `加速持ち` | `inherited_effects.acceleration_mps2 != null` のみ残す |
| `スタミナ回復` | `inherited_effects.stamina_recovery_pct != null` のみ残す |
| `速度持ち` | `inherited_effects.target_speed_mps != null` のみ残す |
| `コスト低め` | `inherited_effects.cost_pt <= 150` のみ残す |

上記に該当しない自由記述の制約は、意味を解釈してルールを適用する。

### ステップ4: 推奨キャラの選出

各脚質ごとに、スコア降順で以下を選出する。**同一キャラは1役割まで（重複禁止）**:

| 役割 | 選出基準 |
|-----|---------|
| 親1 | スコア1位 |
| 祖父母11 | スコア3-6位（親1と異なるキャラ） |
| 祖父母12 | スコア3-6位（親1と祖父母11と異なるキャラ） |
| 親2 | スコア2位（親1と異なるキャラ） |
| 祖父母21 | スコア3-6位（親2と異なるキャラ） |
| 祖父母22 | スコア3-6位（親2と祖父母21と異なるキャラ） |

スコアが同点の場合は `inherited_effects.cost_pt` が低い方を優先する。
候補が4件に満たない場合は「候補不足」と記載する。

### ステップ6: ファイル保存

`output/` ディレクトリが存在しない場合は作成する。

ファイルパス: `output/{レース場}_{路面}_{距離}m_{race_mode}.md`

例: `output/東京_芝_2400m_チャンミ.md` / `output/東京_芝_2400m_LoH.md`

保存後、以下を報告する:
- 保存ファイルパス
- 各脚質の親1（スコア1位）のキャラ名とスキル名
- ユーザー制約で絞り込みを行った場合はその旨

## エラー処理

| 状況 | 対応 |
|-----|-----|
| racetrack.json または trainee.json が存在しない | 処理中止・対応スキルの実行を案内 |
| 指定コースが見つからない | 同会場の存在コース一覧を提示して中止 |
| 全脚質で候補が4件未満 | 不足分を「候補不足」と記載して保存を継続 |
| 制約適用後に候補がゼロ | 警告を出して制約なしの推奨に切り替え |
