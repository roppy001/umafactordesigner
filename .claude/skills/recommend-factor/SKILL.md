# 因子構成推奨

`references/trainee.json` と `references/racetrack.json` を読み込み、
指定のレース・距離・脚質に対する推奨親・祖父母構成を `output/` に出力する。

## 入力パラメータの読み取り

ユーザーの発言から以下を抽出する:

| パラメータ | 例 | 必須 |
|-----------|---|------|
| レース場 | `東京` | ✅ |
| 路面 | `芝` | ✅ |
| 距離（m） | `2400` | ✅ |
| レースモード | `チャンミ` または `LoH` | ✗（デフォルト: `チャンミ`） |
| 制約条件 | `親は加速持ちにしたい` | ✗ |

距離が省略された場合はユーザーに確認する。
路面が省略された場合は `芝` をデフォルトとしてよい。
レースモードが省略された場合は `チャンミ`（9頭立て）をデフォルトとする。

レースモードからフィールドサイズを導出する:

| race_mode | field_size |
|-----------|-----------|
| `チャンミ` | 9 |
| `LoH` | 12 |

## 実行手順

### ステップ1: 参照データの読み込み

`references/racetrack.json` と `references/trainee.json` を読み込む。
どちらかのファイルが存在しない場合は処理を中止し、対応するスキルを先に実行するよう案内する。

- racetrack.json がない → 「レース場情報を保存してください」と案内
- trainee.json がない → 「継承固有の情報を保存してください」と案内

### ステップ2: 対象コースの特定

`racetrack.json` から `venue`・`surface`・`distance_m` が全て一致するコースを検索する。

- 一致するコースが見つからない場合: 同じ `venue` と `surface` で存在するコース一覧を距離順に提示して処理を中止する
- 一致するコースが複数ある場合: 全て対象とする（通常は1件）

### ステップ3: スキルのスコアリング

`trainee.json` の全スキルに対して、以下の観点でスコアを付ける。
スコアは加算方式とし、各脚質ごとに独立して計算する。

#### 3-0. rank_range の参照（レースモード対応）

`activation_conditions.rank_range` はモード別に事前格納済み。`race_mode` に対応するキーを直接参照する:

```
effective_rank_range = rank_range?.[race_mode]   // "チャンミ" or "LoH"
```

- `rank_range` が `null` の場合: 順位条件なし（除外しない）
- `effective_rank_range[0] > field_size` の場合: このフィールドでは発動不能 → **スコア 0 点で候補除外**
- `effective_rank_range[1]` が null の場合: 上限なし扱い（除外しない）

例（race_mode = "チャンミ", field_size = 9 の場合）:
- `{ "チャンミ": [7, 9], "LoH": [7, 12] }` → 実効 `[7, 9]`（有効・後方スキル）
- `{ "チャンミ": null, "LoH": [10, 12] }` → 除外（9頭立て非対応）
- `{ "チャンミ": [1, 3], "LoH": [1, 3] }` → 実効 `[1, 3]`（変化なし・前方スキル）

**rank_multiplier の計算:**

スキルが除外されない場合（`rank_range` が null、または有効な `effective_rank_range` を持つ場合）、以下のように `rank_multiplier` を決定する。この値は 3-1〜3-5 で算出した合計スコアに最後に乗算する。

```
rank_range が null の場合:
  rank_multiplier = 1.0

rank_range が非 null かつ有効な場合:
  range_min = effective_rank_range[0]
  range_max = effective_rank_range[1] ?? field_size   // null は field_size で代替
  prob_table = 上記「順位確率分布表」の race_mode × 対象脚質の行
  rank_multiplier = sum(prob_table[range_min], prob_table[range_min+1], ..., prob_table[range_max])
```

- 対象脚質: 現在スコアリング中の脚質（逃げ / 先行 / 差し / 追込）を使用する
- `running_styles` が `[]`（脚質不問）のスキルも、スコアリング対象脚質の行を参照する
- 確率分布表がすべて 0.00 の場合（ひな形未入力）: `rank_multiplier = 1.0` として扱い、乗算をスキップする

例（先行・チャンミ・rank_range = [3, 4]、表の先行3位=0.25・4位=0.25 の場合）:
```
rank_multiplier = 0.25 + 0.25 = 0.50
最終スコア = (3-1〜3-5 の合計点) × 0.50
```

#### 3-0.5. 発動タイミングの計算（activation_m）

各スキルの発動タイミングを距離（m）で計算する。**ランダム発動スキルは発動区間の真ん中で発動するものとして計算する。**

```
① distance_pct_min と distance_pct_max が両方 null でない場合:
     activation_m = distance_m × (distance_pct_min + distance_pct_max) / 2 / 100

② distance_pct_min のみ非 null（distance_pct_max が null）の場合:
     activation_m = distance_m × distance_pct_min / 100

③ 両方 null（phase のみ）の場合:
     "序盤"  → activation_m = distance_m ×  1/12   （0/6〜1/6 の中央）
     "中盤"  → activation_m = distance_m ×  5/12   （1/6〜4/6 の中央）
     "終盤"  → activation_m = distance_m ×  9/12   （4/6〜5/6 の中央）
     "最終盤" → activation_m = distance_m × 11/12   （5/6〜6/6 の中央）
     null    → activation_m = null（遅れ計算不可）
```

phase_detail が設定されている場合は、以下のように activation_m を上書きする（③ の phase 中央点より優先）:

| phase_detail | 対応区間 | activation_m |
|-------------|---------|-------------|
| `"始め"` | 終盤の前半1/2（16/24〜18/24） | `distance_m × 17/24` |
| `"始め早め"` | 終盤の前半1/4（16/24〜17/24） | `distance_m × 33/48` |
| `"残りXXXm"` | 残り距離固定 | `distance_m - XXX` |

#### 3-1. 脚質フィルター（必須条件）

`activation_conditions.running_styles` が空配列 `[]` または対象脚質を含む場合のみスコア計算対象とする。
含まない場合はその脚質での推奨候補から除外する。

#### 3-2. delay_multiplier の計算（終盤加速スキルのみ）

**対象**: `acceleration_mps2 != null` かつ `phase == "終盤"` のスキル

終盤加速スキルは終盤開始直後に発動するほど効果的であるため、発動タイミングが遅いほどスコアを減衰させる。

```
late_start_m = distance_m × 4/6      // 終盤開始地点

delay_m = max(0, activation_m - late_start_m)
           // activation_m が null の場合は delay_m = 0 として扱う

delay_multiplier = max(0.0, 1.0 - delay_m / 120.0)
   // 遅れ 0m → delay_multiplier = 1.0（減衰なし）
   // 遅れ 120m → delay_multiplier = 0.0（スコア 0）
```

終盤加速以外のスキルは `delay_multiplier = 1.0`（乗算なし）とする。

**phase_detail 別の delay_m 計算例（distance_m = 2400m の場合）:**

| phase_detail | activation_m | delay_m | delay_multiplier |
|-------------|-------------|---------|-----------------|
| null（phase "終盤" のみ） | 2400 × 9/12 = 1800m | 1800 − 1600 = 200m | max(0, 1 − 200/120) = 0.0 |
| `"始め"` | 2400 × 17/24 = 1700m | 1700 − 1600 = 100m | max(0, 1 − 100/120) ≈ 0.17 |
| `"始め早め"` | 2400 × 33/48 = 1650m | 1650 − 1600 = 50m | max(0, 1 − 50/120) ≈ 0.58 |

#### 3-3. スコアの計算

各スキルの最終スコアを以下の3コンポーネントの合計として算出する。各コンポーネントは、発動条件（rank_range・脚質フィルター）を満たした場合のみ計算する（満たさない場合は 0）。

**最終速度スコア**（`target_speed_mps != null` かつ `phase == "最終盤"` の場合）:
```
最終速度スコア = target_speed_mps × duration_sec × rank_multiplier
```

**最終加速スコア**（`acceleration_mps2 != null` かつ `phase == "終盤"` の場合）:
```
最終加速スコア = acceleration_mps2 × duration_sec × rank_multiplier × delay_multiplier
```

**中盤/序盤速度スコア**（上記2条件のいずれにも該当しないスキルの場合）:
```
effect_value = target_speed_mps が非 null の場合はその値、null の場合は stamina_recovery_pct
中盤/序盤速度スコア = effect_value × duration_sec × rank_multiplier
   // effect_value が null の場合は 0
```

**最終スコア**:
```
最終スコア = 最終速度スコア + 最終加速スコア + 中盤/序盤速度スコア
```

ほとんどのスキルでは1コンポーネントのみが非ゼロとなる。

### ステップ4: 推奨キャラの選出

各脚質ごとに以下の手順で6役割を選出する。**同一キャラは1役割まで（重複禁止）**。

#### 4-1. スキルタイプ分類

脚質フィルター・rank_range フィルターを通過した全スキルを以下の4タイプに分類する（上から順に評価し、最初に一致したタイプを適用）:

| タイプ | 判定条件 |
|-------|---------|
| 進化固有 | `category == "進化固有"` |
| 終盤加速 | `effects.acceleration_mps2 != null` かつ `phase == "終盤"` |
| 終盤速度 | `effects.target_speed_mps != null` かつ `phase == "最終盤"` |
| 序盤/中盤 | 上記以外（`phase` が `"中盤"` / `"序盤"` / `null`、効果種別問わず） |

#### 4-2. タイプ別 top4 の選出（candidate pool の構築）

各タイプからスコア降順で上位4件を選出し、**candidate pool**（最大16件）を構築する:

| タイプ | 選出数 |
|-------|-------|
| 進化固有 | 上位4件（存在する場合のみ） |
| 終盤加速 | 上位4件 |
| 終盤速度 | 上位4件 |
| 序盤/中盤 | 上位4件 |

- 各タイプが4件未満の場合はその件数のみ選出する
- candidate pool はこれ以降の全選出の母集団となる（pool 外のスキルは推奨対象外）

#### 4-3. Top10・推奨因子構成の選出

candidate pool の全件をスコア降順にソートし:

1. **推奨スキル Top 10**: 上位10件（10件未満の場合は全件）
2. **推奨因子構成（6役割）**: 以下の手順で選出する（**同一キャラは1役割まで**）

**親スロットの選出（親1・親2）:**

タイプ優先順位（進化固有 → 終盤加速 → 終盤速度 → 序盤/中盤）に従い、各タイプ内でスコア降順に選出する:

1. 優先順位が最も高い候補を **親1** に割り当てる
2. 次に優先順位が高い候補（親1と異なるキャラ）を **親2** に割り当てる
3. 同一タイプ内に複数候補がある場合はスコア降順で選ぶ

**祖父母スロットの選出（バランス制約あり）:**

親1・親2を除いた残りの candidate pool からスコア降順で4枠を選出する。以下のバランス制約を適用する:

- バランス集計対象: **終盤加速・終盤速度・序盤/中盤** の3タイプ（進化固有は除外）
- 6役割全体で各タイプが均等になるよう選出する（理想: 各タイプ2個）
- **同一タイプは6役割全体で最大3個**

```
type_count = { 終盤加速: 0, 終盤速度: 0, 序盤/中盤: 0 }

// 親スロットのタイプをカウント（進化固有は除外）
for each parent in [親1, 親2]:
  if parent.type != "進化固有": type_count[parent.type] += 1

// 祖父母4枠を1枠ずつ選出
for i in 1..4:
  // type_count が最小かつ count < 3 のタイプを持つ候補をスコア降順で選ぶ
  // 複数タイプが同数の場合: 終盤加速 → 終盤速度 → 序盤/中盤 の順を優先
  // 制約を満たす候補がない場合: バランス制約を緩和してスコア降順で選出（警告を出す）
  割り当て; type_count[candidate.type] += 1
```

**祖父母の役割割り振り（選出後）:**
- **祖父母11・祖父母12**: 親1と異なるキャラから選んだ2枠
- **祖父母21・祖父母22**: 親2と異なるキャラから選んだ2枠
- 4枠の中で上記両方を満たせない場合はスコア順で次点を当てて「候補不足」と記載する

スコアが同点の場合は `effects.cost_pt` が低い方を優先する。
候補が4件に満たない場合は「候補不足」と記載する。

#### 推奨スキル Top 10 の生成ルール

各脚質のスコアリング結果（ステップ3）を降順にソートし、上位10件を「推奨スキル Top 10」として出力する。

| 列名 | 内容 |

| キャラ | `full_name` |

- `parent_evolution.is_evolution_trigger = true` のキャラには「⚠ 親進化トリガー」と記載
- `category = "進化固有"` のスキルには「（進化固有）」と記載
- ユーザー制約で絞り込まれた候補には「（制約一致）」と記載

### ステップ6: ファイル保存

`output/` ディレクトリが存在しない場合は作成する。

ファイルパス: `output/{レース場}_{路面}_{距離}m_{race_mode}.md`

例: `output/東京_芝_2400m_チャンミ.md` / `output/東京_芝_2400m_LoH.md`

保存後、以下を報告する:
- 保存ファイルパス
- 各脚質の親1（スコア1位）のキャラ名とスキル名
- ユーザー制約で絞り込みを行った場合はその旨

## エラー処理

| 状況 | 対応 |
|-----|-----|
| racetrack.json または trainee.json が存在しない | 処理中止・対応スキルの実行を案内 |
| 指定コースが見つからない | 同会場の存在コース一覧を提示して中止 |
| 全脚質で候補が4件未満 | 不足分を「候補不足」と記載して保存を継続 |
| 制約適用後に候補がゼロ | 警告を出して制約なしの推奨に切り替え |
