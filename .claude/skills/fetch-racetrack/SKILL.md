# レース場情報取得

U-toolsから全107コースのレース場情報を取得し、`references/racetrack.json` に保存する。

## 実行手順

### ステップ1: コースID一覧の取得

`https://xn--gck1f423k.xn--1bvt37a.tools/race/tracks` を以下のプロンプトでWebFetchする:

> ページ内に存在する全コースのリンク（href）を列挙してください。
> `/race/courses/` で始まるリンクを全て抽出し、以下の形式でリストアップしてください:
> - コースID（数値部分）, レース場名, 距離, 路面
> 全てのコースを漏れなく含めてください。

取得したコースIDを全て記録する。IDが1件も取得できない場合は処理を中止してエラーを報告する。

### ステップ2: 各コースの詳細取得（並列処理）

全コースIDに対して `https://xn--gck1f423k.xn--1bvt37a.tools/race/courses/{id}` を5件ずつ並列でWebFetchする。

各ページから以下のプロンプトで情報を抽出する:

> このレース場ページから以下の全フィールドを抽出してJSON形式で返してください:
> - レース場名（会場名）
> - 距離（メートル数）
> - 路面（芝 / ダート）
> - 回り方向（右 / 左）
> - 坂情報: 各坂について「上り/下り」「開始位置(m)」「終了位置(m)」「勾配(%)」
> - コーナー情報: 各コーナーについて「番号」「開始位置(m)」「終了位置(m)」
> - 直線情報: 各直線について「名称」「開始位置(m)」「終了位置(m)」「長さ(m)」
> - 適正スキル名の一覧（例: 右回り, 根幹距離, 中山レース場 など）
> 値が存在しない・不明な場合は null を返してください。

### ステップ3: データの整形

#### distance_category の判定

距離（`distance_m`）に基づいて以下のルールで判定する:

| 距離範囲 | カテゴリ |
|---------|---------|
| ≤ 1400m | `"短距離"` |
| 1401〜1800m | `"マイル"` |
| 1801〜2400m | `"中距離"` |
| ≥ 2401m | `"長距離"` |

#### phases の計算

ウマ娘の標準フェーズ定義に基づき、`distance_m` から各フェーズを計算する:

| フェーズ | 開始(m) | 終了(m) |
|---------|--------|--------|
| `early` | 0 | `distance_m / 6`（小数切り捨て） |
| `middle` | early終了 | `distance_m * 2 / 3`（小数切り捨て） |
| `late` | middle終了 | `distance_m * 5 / 6`（小数切り捨て） |
| `last_sprint` | late終了 | `distance_m` |
| `position_keep` | 0 | `distance_m * 5 / 12`（小数切り捨て） |

例（1200m）:
- early: 0〜200、middle: 200〜800、late: 800〜1000、last_sprint: 1000〜1200、position_keep: 0〜500

#### slopes の整形

- `type`: 上り坂 → `"uphill"`、下り坂 → `"downhill"`
- `gradient_pct`: 数値（上りは正値、下りは負値）
- 坂情報が存在しない場合は空配列 `[]`

#### corners の整形

- `number`: コーナー番号（整数）
- `start_m` / `end_m`: 区間（m）
- コーナー情報が存在しない場合は空配列 `[]`

#### straights の整形

- `name`: 直線名称（例: `"直線1"`）
- `start_m` / `end_m`: 区間（m）
- `length_m`: 長さ（m）
- 直線情報が存在しない場合は空配列 `[]`

### ステップ4: JSONの組み立て

全コースのデータを以下のスキーマに従ったJSON配列として組み立てる:

```json
[
  {
    "course_id": "10501",
    "venue": "中山",
    "distance_m": 1200,
    "surface": "芝",
    "direction": "右",
    "distance_category": "短距離",
    "phases": {
      "early":         { "start_m": 0,    "end_m": 200  },
      "middle":        { "start_m": 200,  "end_m": 800  },
      "late":          { "start_m": 800,  "end_m": 1000 },
      "last_sprint":   { "start_m": 1000, "end_m": 1200 },
      "position_keep": { "start_m": 0,    "end_m": 500  }
    },
    "slopes": [
      { "type": "downhill", "start_m": 0,    "end_m": 200,  "gradient_pct": -1.5 },
      { "type": "uphill",   "start_m": 1025, "end_m": 1135, "gradient_pct": 2.0  }
    ],
    "corners": [
      { "number": 3, "start_m": 300, "end_m": 650 },
      { "number": 4, "start_m": 650, "end_m": 890 }
    ],
    "straights": [
      { "name": "直線1", "start_m": 890, "end_m": 1200, "length_m": 310 }
    ],
    "skill_aptitudes": ["右回り", "根幹距離", "中山レース場"]
  }
]
```

コースIDの昇順で並べる。

### ステップ5: ファイル保存

組み立てたJSON配列を `references/racetrack.json` に書き込む。
既存の内容は全て上書きする。

保存後、以下を報告する:
- 保存したコース件数（合計）
- レース場別の内訳（例: 中山 6コース、東京 5コース、…）
- エラーや取得できなかったコースがあれば一覧表示

## エラー処理

- WebFetchが失敗したコースIDはスキップし、最後にまとめて報告する
- 座標・勾配情報が読み取れない場合は該当フィールドを `null` にして処理を継続する
- ステップ1でIDが1件も取得できない場合は処理を中止してエラーを報告する
